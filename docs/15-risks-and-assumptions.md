# Risks And Assumptions

## Product Risks

- Users may not manually add enough knowledge before integrations exist.
- Buyers may compare the product to tools they already own.
- AI trust depends on source quality and citation clarity.
- The first dashboard must feel useful despite limited analytics.

## Technical Risks

- Tenant isolation bugs can cause severe data leakage.
- Document extraction quality varies by file type.
- Vector retrieval may miss key facts without strong chunking and metadata.
- Prompt injection can manipulate answers if sources are treated as instructions.
- Integration syncs can create duplicate or stale knowledge.

## Validation Plan

- Interview 10-15 SMB leaders and managers.
- Demo source-cited answers using manually entered project data.
- Test whether users trust "not enough information" responses.
- Measure time saved finding decisions and latest documents.
- Validate willingness to connect documents and project systems.
